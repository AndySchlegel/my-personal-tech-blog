OIDC Federation — Zero AWS Keys

Problem

Langlebige AWS Access Keys in GitHub Secrets = Sicherheitsrisiko + Rotationsaufwand

Loesung

GitHub OIDC Provider in AWS + IAM Role mit Trust Policy. Kurzlebige Credentials per Workflow-Run.

Ergebnis

Nur 4 GitHub Secrets: AWS_ROLE_ARN, DB_PASSWORD, TELEGRAM_BOT_TOKEN, TELEGRAM_CHAT_ID

Infrastruktur-Security

S3 Public Access komplett blockiert (nur CloudFront OAC)
RDS in Private Subnets (SG-to-SG Rules)
EKS Secrets verschluesselt (KMS at rest)
IRSA fuer Pod-Level IAM (Least Privilege)
TLS 1.2+ auf CloudFront erzwungen
Keine AWS Credentials in CI/CD (OIDC)

Key Design Decisions

  • ALB target-type: ip — VPC CNI routet direkt zu Pod IPs
  • Spot Instances — t3.medium + t3a.medium (~70% guenstiger)
  • nginx Reverse Proxy — ALB → nginx → /api/* → Backend
  • kubectl create secret --dry-run — echte Werte, nie hardcoded
  • Liveness + Readiness — HTTP /health (Backend), TCP + HTTP (Frontend)
  • DB Init als Job — idempotentes Schema + Seed gegen RDS

Lessons Learned

34 dokumentierte Learnings — jedes mit konkretem Problem und Loesung

#1 Security Scanning ab Tag 1

Security ist kein Feature das man spaeter hinzufuegt — es ist das Fundament auf dem man baut.

#17 Dynamic Terraform Outputs

Pipeline liest Infra-Werte aus Terraform State — kein manuelles Secret-Update nach Destroy+Apply.

#21 Circular Dependency Deadlock

OIDC/IAM Zirkularabhaengigkeit. Fix: lifecycle ignore_changes + Wave 0 Pre-Step.

#23 EKS Cluster Security Group

Managed Nodes nutzen NICHT die Custom SG. EKS erstellt eine eigene Cluster SG fuer alle Nodes.

#26 ALB Controller Orphans

K8s Controller erstellt ALBs ausserhalb von Terraform. Cleanup BEFORE EKS Destroy noetig.

#30 Full Lifecycle Verified

Reproduzierbarkeit beweist man durch den vollen Zyklus. Jede Iteration deckt neue Edge Cases auf.

#31 Telegram ueber SES

Native fetch statt npm-Paket. Non-blocking Fire-and-Forget. Notification-Fehler blockieren nie die Antwort.

#32 Dual-Track Hosting

EKS fuer Showcase ($143/Sprint), Lightsail fuer Dauerbetrieb ($5.50). Kubernetes-Expertise zeigt sich daran, wann man es NICHT nutzt.
